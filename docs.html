BizShow Docs



SetUp

Bizshow is self-contained so that you can quickly prototype a one-page html document of an api or app. The fonts, js, and css files are in the assets sub-folder. The prism componenets that define code highlighting formating per language are in the components sub-folder. The same exact demo is the same exact layout so you can hit-the-ground running and create that one page web doucment in minutes following this demo example.

  • In Layouts
    • layout-one.html
    • favicons and favicon configs
    • assets
      • css
      • fonts
      • images
      • js

Each layout file has its own layout-name-init.js and layout-name-style.css file.

CDN urls of the components are listed in the last section so that you can fine tune according to your bandwidth performance requirements.

Banner Transitions

I use a combination of bounceJS, animateCSS, lettering, and textillate plugins so that there is less code to write both in CSS and JS. Side Note: There is an online version of BounceJS:

BounceJsONline

You will use that to generate the CSS keyframes if you do not like using javascript to do it. The javascript I used in layout-one is:

Javascript


                var bounce = new Bounce();
                bounce
                  .translate({
                    from: { x: 0, y: 170 },
                    to: { x: 0, y: 0 },
                    duration: 12600,
                    stiffness: 4
                  })
                  .scale({
                    from: { x: 1, y: 1 },
                    to: { x: 0.1, y: 2.3 },
                    easing: "sway",
                    duration: 19800,
                    delay: 65,
                    stiffness: 2
                  })
                  .scale({
                    from: { x: 1, y: 1},
                    to: { x: 5, y: 1 },
                    easing: "sway",
                    duration: 19300,
                    delay: 30,
                  })
                 .applyTo(document.querySelectorAll(".hero-phrase"));

Basically, I create an empty var called bounce that is set equal to a new Bounce instance than I define the animation settings. And the last step is to set it to trigger on an html element via the apply-to statement.

There is a transitions tutorial being developed as an upcoming update that will step you through DIY-wise in coming up with your own transitions and even using javascript callbacks to tie-them togehter in more complex and cool transitions stuff. The next banner transition is the texillate one which uses both animateCSS and lettering as dependencies. Let me step you through the one applied to the layout-one hero-title:

Javascript


                $('.hero-title').textillate({
                  type: 'word',
                  in: {




                    // set the delay between each character
                    delay: 1250,

                    // set to true to animate all the characters at the same time
                    sync: false,

                    // randomize the character sequence
                    // (note that shuffle doesn't make sense with sync = true)
                    shuffle: false,


                  },


                });

We are telling textillate to use lettering in the word mode and than for the in-effects we are setting the delay, sync, and shuffle.

In our HTML on the hero-title ellement we have:

HTML


                data-in-effect="bounceInLeft"

The last bit is the color transition which is completed in CSS:

CSS


                @keyframes toBlueGrey {
                  0% {
                    color: #e8eaf6;
                  }
                  60% {
                    color: #c5cae9;
                  }
                  100% {
                    color: #9fa8da;
                  }
                }
                @keyframes toGrey{
                  0% {
                    color: #e3f2fd;
                    font-size: 1.25rem;
                  }
                  60% {
                    color: #bbdefb;
                    font-size: 1.5rem;
                  }
                  100%{
                    color: #90caf9;
                    font-size: 2.25rem;
                  }
                }
                h1.hero-title span.word1{
                    animation-name: toBlueGrey;
                    animation-duration: 18s;
                    animation-fill-mode: forwards;

                }
                h1.hero-title span.word2{
                  animation-name: toGrey;
                  animation-duration: 18s;
                  animation-fill-mode: forwards;
                  margin-left: -0.55rem;
                }

Some notes about this CSS code. In my htnl h1 title I kept the words sperate as I wanted to be able to use lettering's generated spans to color the title. So I use a negative margin left to adjust for that quirk.

Although its set to the hero-title element the animation will not complete until the textiallate js animation has executed first. That is one of the reasons I am completing a transitions tutorial using AOS, bounceJS, lettering, animateCSS, and textillate as I want you to have the power to change in what sequence things get executed as opens the doors to more powerfull attention grabbing tranitions.

Now onward to the AOS transtions

AOS Transitions

Velocity JS generally is somewhat hard to understand as far as doing transitions. So the AOS plugin is used instead as it makes it a breaze to do transitions. Let me show how eas it is. First lets initialaize the AOS plugin:

Javascrrript


                AOS.init();

Every AOS powered transition is detailed via data attributes, for examplee the promo block has:

HTML


               data-aos="flip-left" data-aos-delay="100" data-aos-anchor=".brand"

The anchor happens to be the card extra class of brand, data-aos specifices the animation to be used and the data-aos-delay sets the delay. By setting up on each individual promo card it will be correctly triggred via a user scroll both on the desktop and mobile.

CDN Urls

First when using Google's CDN for fonts do not just insert the new css imports in the html head as you still need to delete the fonts.css import as it points to the local copy of the font files.

Okay, the icon fonts first andd rember to remove the local font-awesome and material-icons css imports after you insert these two new css imports into the html head of the layout variant you aare using:
